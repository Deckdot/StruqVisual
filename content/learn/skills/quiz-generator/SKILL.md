# Skill: quiz-generator

## Purpose
Generate high-quality quiz questions for Struq Learn lessons.

## Quality Rules
- Questions test real understanding, not trivia
- Wrong answer choices should be plausible — not obviously wrong
- Explanations must add genuine insight, not just restate the answer
- Short-answer questions should be used for open-ended reasoning
- Multiple-choice is best for conceptual distinctions and tradeoffs
- Single-choice is best for "which is correct" factual questions
- Points should reflect difficulty: conceptual reasoning = 3–5 pts, recall = 1–2 pts
- Avoid questions with "all of the above" or "none of the above"

## Domain Focus
Questions should specifically test:
- Practical application of concepts (not just definitions)
- Tradeoff recognition (knowing WHEN to use something, not just WHAT it is)
- Pattern recognition in code examples and architecture diagrams
- Workflow sequencing (what comes before/after what)
- Anti-pattern identification (spotting what NOT to do)
