# Skill: workflow-breakdown

## Purpose
Break down AI-native builder workflows into concrete, teachable steps.

## Domain Focus
Workflows relevant to Struq Learn:
- vibecoding sprint workflow
- prompt library design workflow
- context file assembly workflow
- agentic task delegation workflow
- AI-assisted debugging workflow
- skill pack creation workflow
- lesson generation and curation workflow

## Quality Rules
- Each step must be actionable, not conceptual
- Each step should include what tool/technique is used
- Pitfalls are more valuable than the steps themselves — be specific
- Common mistakes should be patterns, not one-off errors
- Best practices should be observable behaviors, not principles
- Tradeoffs should name the actual cost of each choice

## Step Quality Checklist
- Step title: imperative verb + specific object ("Map out context boundaries", not "Think about context")
- Step description: what you do, why, and how you know it's done
- Tool/technique: name the actual tool (Claude Code, Cursor, Windsurf, etc.)
- Pitfall: the most common mistake at this step and its consequence
